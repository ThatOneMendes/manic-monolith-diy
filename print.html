Introduction

Welcome to the Manic Monolith: Do It Yourself book! This book will teach you the basics of the level editor, tricks and tips, and the use of ALL the objects in the game!

note

The book and the game are still in heavy developement, expect some things to be slightly out of date...

Editor UI

One of the first thing you will have to learn is how to use the level editor UI, so this subchapter will walk you through all the stuff you need to know before you start using the level editor.

The options bar

One of the core elements of the UI, its simple, but you will be using this very often. Its composed of various buttons each with a different function.

Hide button

Its the ">" button, it just hides the options bar so that you can have more screen space.

Playtest

When pressed it teleports you to the main game where your level is instantly loaded after the loading screen finishes

You can't playtest your level if your level settings are invalid

If you haven't saved your level, the playtest button will give you a warning and do nothing.

Objects

Another very important element of the level editor, when pressed, a vertical bar will appear at the left side of the screen, displaying all of the objects you can place in a level

Clicking on an object puts it in your mouse cursor, from there, drag your mouse at the place you want to put the object an simply click with the left mouse button, ðŸŽ‰ You have placed your first object!

Level Settings

Its where you tweak your level behavior, like the amount of points needed for each rank, level track ID, level name ETC...

Object Tree

When clicked, an view of all the objects in the current level will show in a tree like hierarchy. Clicking on an object will select it.

Save/Load Level

Allows you to save, load and erase your levels. You can also:

  • Save a level as new level (you won't overwrite any levels you already got)
  • Load a new empty level

Current View Toggle

It changes the current view of your level when clicked. The order goes by (it loops back to the first view mode)

  1. Level Mode:
    • Its the default mode, you can only see and edit the main world (where Mart spawns)
  2. Secret 1-3 Mode:
    • All those are like the Level Mode, but you can only see and edit their respective secrets.

Property Editor

Another very important element of the level editor, the property editor allows you to change an object properties, like size, position, color, etc...

Keybinds

Knowing all keybinds is not needed for using the level editor, but they can boost your productivity by a lot! Except the delete key you are going to use it sooner or later

Delete key

Deletes the selected Object, be wary that some object can't be deleted.

Ctrl-Z key combination

Undoes the last action, only some actions can be undone (this is going to change in the future)

F key

If an Object is selected, move the camera to that Object's position (if it has one)

F4 Key

Changes the position increment, in studs.

Ctrl-F4 key combination

Changes the rotation increment, in degrees.

Q key

If in position edit mode, pressing Q will change the positioning mode from local to world and vice-versa.

Ctrl-Q key combination

Hides/shows the position handle.

R Key

If in rotation edit mode, and if the object being edited has an Attachment child named "Pivot", this will toggle between relative rotation and world rotation (relative rotation rotates the object relative to the Pivot Attachment)

Ctrl-R key combination

Hides/shows the rotation handle.

U Key

Unfocuses an Object.

Objects

THE most important feature in the entire level editor, how are you gonna make a level with no objects?

Thats right, you can't!

This chapter will have ALL the objects and their properties, this is intended to be used as a documentation for level creators.

Attachment

An Attachment is used to store a position relative to its parent, for example, every Lap 2 object will have an Attachment as its child, said attachment points to the position Mart is going to be teleported when he touches the Lap 2 object.

Properties

Positioning

This property can only be edited with the position/rotation handle. Its value will be where the Attachment is relative to its parent

Placing

When you attempt to place an Attachment in the level, your mouse icon will change and elegible objects will gain a orange-ish outline on them, click on an elegible object and the attachment will be placed as a child of that object.

Birby

Birby is an enemy with no behaviour, its just static object that can be killed.

Properties

Positioning

This property can only be edited with the position/rotation handle. Its value is simply where Birby is in the level.

Color Value

This object has one purpose: hold a color value

Properties

Value

Any valid color.

Placement

Upon selecting to place a Color Value, your mouse icon will change, hovering your mouse over elegible objects will add a white outline to them, clicking on an elegible object places the Color Value as a child of said elegible object.

Breakable Block (weak)

Its a solid block which breaks when Mart collides with it at reasonably slow speeds. Grabbing breaks them, so does groundpounding/divebombing on them.

Properties

Size

Determines this object's size

note

The block's durability won't change with size! Its best to just have a long sequence of individual breakable blocks if you thought of doing that!

Positioning

This property can only be edited with the position/rotation handle. The value corresponds to where the Breakable Block (weak) is in the level

Color

Determines this object's color.

Material

Determines this object's appearance

Reflectance

An property which defines how reflective this object is. 1 is totally reflective, and 0 doesn't reflect anything at all.

Shape

Determines this object's shape. Those shapes are:

  1. Ball
    • A spherical shape.
  2. Block
    • A block shape.
  3. Cylinder
    • A cylinder shape.
  4. Wedge
    • A wedge shape with a slope on one side.
  5. Corner Wedge
    • A wedge shape with slopes on two sides.

Can Collide

Sets if this object can collide with others, if an object is unanchored and has the Can Collide value set to false, the object will fall through the world as soon as the level is loaded.

Linear velocity

A Vector3 value that determies the object's linear velocity.

When either the X, Y or Z values are not 0, Mart's velocity will be affected by the object's velocity.

The two most important interactions are as follow but others can happen depending on which values you input.

Angular Velocity

A Vector3 value that determies the object's angular velocity

This one is a little more complicated, and there aren't visual examples of it in action! (yet)

On top of that, Mart can't interact with Angular Velocity, only unanchored objects!

Decal

This object is a static image which is stretched into an object's surface. Not all objects can have Decals applied to them, however. For big levels, try to use Decals when possible instead of Textures, since its more lightweight on the player's device.

Properties

Color

Determines this object's color. Be wary that some colors wont actually change (eg.: black)

Image ID

A roblox Decal ID that the Decal will use.

Transparency

Determines how transparent the object is, this value ranges from 0 to 1. 1 is fully transparent while 0 is fully opaque

Z Index

Determines the depth of the texture relative to others.

Example: Lets say that you have two Decals, and you want Decal A to be in front of Decal B, to solve that, simply set Decal A's Z Index to something higher than Decal B's Z Index.

Face

A property which defines what surface will this Decal be on.

Placement

When you attempt to place a Decal, your mouse icon will change and elegible objects will have a missing texture decal applied to them when you hover your mouse over said objects, clicking on an elegible object will place the decal in the surface your mouse was pointing at.

Gravity Trigger

Gravity Trigger is an object which changes the current world's gravity when mart touches it, restoring the world's gravity when contact with the trigger stops.

Remarks

All Gravity Triggers have a Number Value child which determines gravity override.

Properties

Size

Determines this object's size

Positioning

This property can only be edited with the position/rotation handle. The value corresponds to where the Gravity Trigger is in the level.

Impulse Trigger

Impulse Triggers are objects which apply a set amount of impulse to any object that touches it.

Remarks

All Impulse Triggers have two children:

  • A Vector 3 Value named "Impulse Direction":
    • This child stores the impulse direction of the trigger, a value of 0, 15, 0 will apply impulse to any object in a upwards direction.
  • A Number Value named "Impulse Force":
    • This child stores the force of the impulse, higher numbers makes this trigger apply stronger impulses on objects.

However you can add those respective objects in order to modify the Impulse Trigger's behavior (remember, you have to add those objects to the Impulse Trigger yourself. They are optional.):

  • A Number Value named "Debounce Timer":
    • This child stores the amount of seconds that the Impulse Trigger needs to apply another impulse (A.K.A: Cooldown)
  • A String Value named "Sound ID":
    • This child stores a sound ID that will be played then the Impulse Trigger gets activated by something.

Properties

Size

Determines this object's size

Positioning

This property can only be edited with the position/rotation handle. The value corresponds to where the Impulse Trigger is in the level.

Secret

An secret object has a Number Value as it child. When Mart touches this object, he gets teleported into the secret tied with the Number Value's Value property. If mart touches this object while already in a secret, he gets sent back to the secret he first went through.

Remarks

This object has a Number Value as its child, this Number Value, stores a value ranging from 1 to 3, this value must be unique between all secrets in the main level.

Properties

Positioning

This property can only be edited with the position/rotation handle. Its value will be where the Secret is in the level.

Sign

Signs have the same properties and behaviour of Studs, but they can have text.

Remarks

Every Sign has 3 children:

  • A String Value named "Text":
    • Contains the text that will be shown when the level is played.
  • A String Value named "Font":
    • Contains the font that will be used for the text in the Sign, if the font doesn't exist, the sign won't render its text, instead it will show: "Unknown font (invalid font will be shown here)!"
    • All valid fonts are listed in this roblox documentation page
  • A Color Value named "Text Color":
    • Contains the color that the text will use when the level is played.

Texture

This object is a dynamic image which repeats in an object's surface. Not all objects can have Textures applied to them, however. For big levels, avoid using too many Textures, since they are heavier to render than Decals.

Properties

Color

This property changes the color of the Texture, be wary that some colors wont actually change (eg.: black)

Offset Studs U

Determines how far the rendered texture is offset on the horizontal axis in studs.

Offset Studs V

Determines how far the rendered texture is offset on the horizontal axis in studs.

Studs/Tile U

Sets the horizontal size, in studs, of the tiled image applied by the Texture.

Studs/Tile V

Sets the vertical size, in studs, of the tiled image applied by the Texture.

Image ID

A roblox Decal ID that the Texture will use.

Transparency

A number which ranges from 0 to 1, in which 1 is fully transparent while 0 is fully opaque

Z Index

Determines the depth of the Texture relative to others.

Example: Lets say that you have two Textures, and you want Decal A to be in front of Decal B, to solve that, simply set Decal A's Z Index to something higher than Decal B's Z Index.

Face

A property which defines what surface will this Decal be on.

Placement

When you attempt to place a Texture, your mouse icon will change and elegible objects will have a missing texture image applied to them when you hover your mouse over said objects, clicking on an elegible object will place the Texture in the surface your mouse was pointing at.

2D Trigger

2D Trigger is an object which changes the camera to 2D mode while Mart is touching it. Every 2D Trigger has a Vector3 Value child which holds the camera axis override

Remarks

All 2D Triggers have a Vector 3 Value child which determines the camera axis override.

warning

There are only two stable axis overrides, those are 1, 1, 0 or 0, 1, 1. All other axis override combinations are unstable. Although unstable, some combinations can have geniuine uses.

Properties

Size

Determines this object's size

Positioning

This property can only be edited with the position/rotation handle. The value corresponds to where the 2D Trigger is in the level.
